# Tinalux 主题系统设计与实现

> 更新时间：2026-03-18  
> 状态：已实现基础主题系统；本文只保留和当前代码一致的说明。

## 当前设计

Tinalux 目前采用的是“全局主题 + typed style token”的主题体系，而不是通用 CSS 引擎。

结构如下：

```text
Theme
  ├─ ColorScheme
  ├─ Typography
  ├─ Spacing
  └─ typed style structs
```

其中 typed style token 包括：

- `ButtonStyle`
- `TextInputStyle`
- `CheckboxStyle`
- `RadioStyle`
- `ToggleStyle`
- `SliderStyle`
- `ScrollViewStyle`
- `DialogStyle`
- `PanelStyle`
- `ListViewStyle`
- `RichTextStyle`

## 已实现能力

### 主题数据结构

`Theme` 当前包含：

- `colors`
- `typography`
- `spacingScale`
- 各控件 style token
- 若干平铺的兼容字段，例如 `background`、`primary`、`text`

入口方法：

- `Theme::dark()`
- `Theme::light()`
- `Theme::custom(...)`

同步方法：

- `syncDerivedTokens()`
- `syncStructuredTokens()`

这意味着主题修改后，既会更新颜色 token，也会刷新控件级 style。

### 主题管理器

`ThemeManager` 当前已支持：

- 注册主题：`registerTheme`
- 切换主题：`setTheme` / `switchTheme`
- 变化监听：`addThemeChangeListener`
- 主题版本号：`version`
- 主题切换动画
- 主题偏好持久化：`saveThemePreference` / `loadThemePreference`

持久化路径支持环境变量覆盖：

- `TINALUX_THEME_PREF_PATH`

### 运行时接线

`UIContext` 会在初始化时：

- 读取 `ThemeManager::currentTheme()`
- 绑定主题变化监听
- 让 runtime state 中的 theme 与 `ThemeManager` 保持同步
- 让主题切换动画能够通过当前 `AnimationSink` 驱动

### 状态样式

状态样式通过 `WidgetState + StateStyle<T>` 实现，支持：

- `Normal`
- `Hovered`
- `Pressed`
- `Disabled`
- `Focused`

这套机制已被多个 style struct 使用。

## 当前边界

下面这些能力不要再写成“已存在”：

- 通用样式继承树
- 任意 Widget 的统一实例级 style 覆盖系统
- 类似 CSS 的选择器
- 主题资源包和完整皮肤系统

更准确的表述应当是：

- 当前已经有完整的主题 token 和 ThemeManager
- 但主要是 typed style 体系，不是通用样式语言

## 适合当前代码的使用方式

### 1. 切换全局主题

```cpp
using namespace tinalux::ui;

ThemeManager::instance().switchTheme("light", true);
```

### 2. 注册自定义主题

```cpp
using namespace tinalux::ui;

Theme ocean = Theme::custom(ColorScheme::custom(tinalux::core::colorRGB(0, 122, 255)));
ocean.setName("ocean");
ThemeManager::instance().registerTheme("ocean", ocean);
ThemeManager::instance().switchTheme("ocean", false);
```

### 3. 监听主题变化

```cpp
const auto listenerId = tinalux::ui::ThemeManager::instance().addThemeChangeListener(
    [](const tinalux::ui::Theme& theme) {
        (void)theme;
        // 触发外部状态更新
    });
```

## 当前剩余工作

- 补齐文档里各个控件对 typed style 的实际覆盖范围
- 明确哪些控件支持实例级 style override，哪些只消费主题默认值
- 如果后续真要做更强的样式系统，应单独设计，而不是误把现有 token 体系写成“CSS 化”
