# Tinalux 开发计划

> 更新时间：2026-03-18  
> 说明：旧版按“阶段一到阶段七”的路线图已经基本失去维护价值。当前更适合按主线问题来安排工作。

## 已完成的主干能力

- 桌面窗口主链路
- OpenGL / Vulkan 渲染
- 后端自动候选与 fallback
- 约束布局与高级布局
- 常见控件库
- 富文本与文本输入
- 主题系统
- 资源管理与异步图片加载
- Android 运行时与宿主接线

## 当前优先级

### P0：构建与验证收敛

- 重新生成并完整构建测试目标
- 修复本地 `ctest -N` 与源码定义的 smoke target 数量不一致问题
- 形成一条可信的桌面 smoke test 验证路径

### P1：Android 真机闭环

- 验证 `android/validation-app`
- 验证 `OpenGL` 默认后端
- 验证显式 `Vulkan` 后端切换
- 验证输入法、剪贴板、pause / resume、surface detach / attach

### P1：Linux/X11 运行时闭环

- 真机验证窗口创建和渲染
- 真机验证 IME 组合态链路
- 记录不同输入法的实际差异

### P2：渲染后端补齐

- 实现 `Metal` 后端
- 让 Apple 平台的 `Backend::Auto` 真正具备 `Metal -> Vulkan -> OpenGL` 的完整候选链

### P2：移动端交互能力

- 触摸语义层
- 手势系统
- 更明确的移动端输入策略

### P3：工具与工程化

- Widget Inspector
- 热重载
- 布局调试工具
- 更系统的性能分析工具

## 暂不建议优先投入的方向

- 继续扩写“完成度百分比”式状态文档
- 在没有真机验证前过度细化 Linux/X11 平台结论
- 在 `Metal` 未落地前宣称三后端都可用

## 里程碑定义

### 里程碑 A

构建目录与测试清单收敛，桌面 smoke tests 可稳定执行。

### 里程碑 B

Android 验证工程完成最小真机闭环，默认后端和显式 Vulkan 都跑通。

### 里程碑 C

Linux/X11 运行时验证完成，平台状态从“代码已接通”升级到“已验证”。

### 里程碑 D

Metal 后端落地，渲染后端矩阵补齐。
