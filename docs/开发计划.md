- OpenGL / Vulkan 渲染
- 后端自动候选与 fallback
- 约束布局与高级布局

- 主题系统

- Android 运行时与宿主接线

## 当前优先级

### P0：构建与验证收敛

- 新增 Windows 桌面 smoke workflow，固化 `syncSkia -> configure -> build -> ctest` 链路
- 新增 Android build scripts smoke workflow，单独守护 `stage / validate-only` 脚本链路
- 为 Windows 桌面 smoke 增加 Skia 源码和 Debug 构建产物缓存，降低冷启动成本
- 把当前已收敛的桌面 smoke test 路径固化到 CI
- 为三条 workflow 明确超时边界，并统一 failure artifact 保留周期
- 为 Windows smoke 补齐 configure / build 阶段的失败现场，减少回溯信息缺口
- 继续收窄无效触发，避免与实际执行链路无关的 CI 运行
- 已将 Linux X11 workflow 的触发边界收敛到 `tina_glfw` 自身和校验脚本
- 已将 Android build scripts smoke 对齐到真实 SDK 模块契约，并收窄到实际依赖文件
- Android build scripts smoke 已补默认 `SdkModuleRoot` 解析覆盖
- Android build scripts smoke 已补 SDK host/sourceSet 与 manifest 基本契约校验
- Android build scripts smoke 已补 `ValidateOnly` 对已 staged SDK 产物的无副作用校验
- Android build scripts smoke 已补 `consumer-rules.pro` 基本契约校验
- Windows 桌面 smoke 已从执行链排除 Android PowerShell smoke，改由独立 Android workflow 守护
- Windows 桌面 smoke 已补入口 / label 边界契约校验，避免脚本与 CMake target 回退到混跑状态
- Windows 桌面 smoke 已收敛 Skia 签名文件改写策略，避免缓存命中后仍重复执行 `gn gen`
- Windows 桌面 smoke 的 Skia build cache 已补同 revision 前缀回退 restore 和 Release 签名缓存
- Windows 桌面 smoke 已补 workflow summary 缓存摘要和 runner 指纹记录，降低 runner 漂移和 cache 命中波动的回溯成本
- Windows 桌面 smoke 已补 configure / build / test 三阶段耗时摘要，便于识别瓶颈落在 configure、编译还是执行阶段
- Windows 桌面 smoke 已补最慢测试 Top 摘要和累计测试耗时记录，便于持续定位桌面回归热点
- Windows 桌面 smoke 已把 metadata 目录独立上传为 artifact，成功和失败场景都可回收 cache / runner / timing 观测结果
- 保持普通 `pwsh` + Ninja + MSVC 环境下的构建稳定性
- 继续观察 Skia 自动构建与 smoke 构建的耦合成本
