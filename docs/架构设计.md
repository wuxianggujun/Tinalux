# Tinalux 架构设计

> 更新时间：2026-03-18  
> 本文只保留和当前源码一致的架构说明，已移除早期草案式内容。

## 总览

```text
              ┌─ Platform ──→ Core
App ──────────┼─ Rendering ─→ Core + Skia
              └─ UI ────────→ Core + Skia
```

关键约束：

- `App` 是唯一的组装层
- `Rendering` 不依赖 `Platform`
- `UI` 不依赖 `Rendering`
- 公共渲染 API 对外暴露的是 `Canvas / RenderContext / RenderSurface / Image`，而不是主要 Skia 类型

## 模块职责

### Core

- 事件类型
- 几何与颜色
- 键码
- 日志

### Platform

- `Window` 抽象接口
- 桌面 `GLFWWindow`
- Android `AndroidWindow`
- 剪贴板、文本输入激活状态、IME 光标矩形、Vulkan surface 平台接口

### Rendering

- `rendering::createContext`
- `createWindowSurface`
- `flushFrame`
- `createRasterSurface`
- 图片创建与加载
- 字体缓存、图片缓存、文本后端

### UI

- `Widget` / `Container`
- 布局系统
- 控件与样式
- 动画调度器
- 资源与图标
- 富文本、文本输入

### App

- `Application`
- `UIContext`
- `BackendPlan`
- Android runtime glue

## 启动与后端选择

当前启动路径的核心逻辑如下：

```text
Application::init(config)
  ├─ initSkia()
  ├─ BackendPlan.reset(config.backend)
  ├─ 逐个尝试 backend candidate
  │   ├─ createWindow(windowConfig)
  │   ├─ createContext(contextConfig)
  │   └─ createWindowSurface(context, window)
  └─ 成功后进入主循环 / pumpOnce
```

候选策略：

- 非 Apple 平台：`Vulkan -> OpenGL`
- Apple 平台：`Metal -> Vulkan -> OpenGL`
- Android 默认配置：`OpenGL`

运行时如果 surface 恢复失败，`Application::tryPromoteNextBackend()` 会尝试下一个候选后端。

## 主循环与运行时状态

`Application` 当前负责：

- 窗口轮询或等待
- 调用 `UIContext::tickAnimations`
- 调用 `UIContext::tickAsyncResources`
- 准备帧并提交
- 同步剪贴板、文本输入激活状态和 IME 光标矩形

`UIContext` 当前负责：

- root / overlay Widget 树
- capture → target → bubble 事件派发
- 焦点管理
- hover 追踪
- 鼠标捕获
- 布局与绘制
- 主题运行时状态
- 动画调度器
- 异步资源回调泵送

## 布局系统

布局仍然采用两阶段模型：

```text
measure(constraints) -> 得到期望尺寸
arrange(bounds)      -> 获得最终位置和大小
```

当前布局能力：

- `VBoxLayout`
- `HBoxLayout`
- `FlexLayout`
  - `direction`
  - `justifyContent`
  - `alignItems`
  - `wrap`
  - `setFlex(child, grow, shrink)`
- `GridLayout`
  - 行列数
  - cell 位置
  - span
- `ResponsiveLayout`
  - 多 breakpoint 选择子布局

## 事件系统

当前事件链路已经稳定在 `UIContext`：

```text
命中测试 -> 事件路径 -> capture -> target -> bubble
```

关键点：

- 键盘事件沿焦点路径传播
- 鼠标事件优先命中 overlay
- `WindowFocus` 会协助清理焦点 / hover 状态
- 文本输入相关事件已并入统一事件体系

## 主题与样式

主题系统由下面几层组成：

```text
Theme
  ├─ ColorScheme
  ├─ Typography
  ├─ Spacing
  └─ typed style structs
```

其中 typed style 包括：

- `ButtonStyle`
- `TextInputStyle`
- `CheckboxStyle`
- `RadioStyle`
- `ToggleStyle`
- `SliderStyle`
- `ScrollViewStyle`
- `DialogStyle`
- `PanelStyle`
- `ListViewStyle`
- `RichTextStyle`

`ThemeManager` 提供：

- 注册主题
- 切换主题
- 主题变化监听
- 切换动画
- 简单持久化

## 资源与异步加载

资源层当前分两层：

- `ResourceManager`
  - 搜索路径
  - 按 DPI 选择 `@2x/@3x`
- `ResourceLoader`
  - 单 worker 线程异步加载图片
  - in-flight 去重
  - 回调回主线程 pump

`UIContext` 每帧会调用 `tickAsyncResources()`，让异步图片结果在 UI 层落地。

## Android 架构

当前 Android 不是单纯预研，而是已经有一条实际接线：

```text
TinaluxActivity / TinaluxSurfaceView
  -> TinaluxRendererHost
  -> TinaluxNativeBridge
  -> Android C ABI
  -> AndroidRuntime
  -> Application
  -> UIContext
```

运行时特性：

- `attachWindow`
- `detachWindow`
- `renderOnce`
- `pause / resume`
- backend 切换

这条链路仍需要真机验证，但架构已从文档方案进入实现阶段。

## 仍然缺失的架构能力

- Metal 渲染后端
- 手势 / 触摸语义层
- 无障碍
- 通用 Inspector / 热重载 / 布局调试工具
