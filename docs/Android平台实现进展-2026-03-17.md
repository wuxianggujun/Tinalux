- 新增 `src/platform/android/AndroidWindow.h`
- 新增 `src/platform/android/AndroidWindow.cpp`
- 新增 `include/tinalux/app/android/AndroidRuntime.h`
- 新增 `include/tinalux/app/android/AndroidNativeBridge.h`
- 新增 `src/app/android/AndroidRuntime.cpp`
- 新增 `src/app/android/AndroidNativeEntry.cpp`
- 新增 `src/app/android/AndroidJniBridge.cpp`
- 新增 `android/host` 宿主侧 Kotlin 骨架
- `WindowConfig` 新增 `android` 原生句柄配置：
  - `nativeWindow`
  - `dpiScale`
- `src/platform/Window.cpp` 增加 Android 平台工厂分支
- `src/platform/CMakeLists.txt` 增加 Android 平台构建接线
- `Application` 新增 `pumpOnce()`，允许外部平台按帧驱动而不是强耦合桌面主循环
- 顶层 `CMakeLists.txt` 现在区分桌面可执行和 Android 原生共享库目标
- Android 构建不再强制依赖 `tina_glfw`

## 当前已实现能力

- 通过 `WindowConfig::android.nativeWindow` 接收外部注入的 `ANativeWindow`
- `AndroidWindow` 构造时执行 `ANativeWindow_acquire()`，析构时执行 `ANativeWindow_release()`
- 当 `graphicsApi == OpenGL` 时：
  - 初始化 `EGLDisplay`
  - 选择窗口配置
  - 创建 `EGLSurface`
  - 创建 OpenGL ES 上下文
  - 调用 `eglMakeCurrent()`
  - 根据 `vsync` 设置 `eglSwapInterval()`
- 提供 `glGetProcAddress()` 供 Rendering 层创建 Skia GL context
- `swapBuffers()` 封装 `eglSwapBuffers()`，失败时记录日志并请求关闭
- 支持 `GraphicsAPI::None`，为后续 Vulkan/纯 Native Window 路径预留无 EGL 包装模式
- `AndroidRuntime` 持有 `Application` 生命周期，负责：
  - 保存 Android 启动配置
  - 在 window attach 时创建 `Application`
  - 在 window detach / app close 时释放渲染资源
  - 通过 `renderOnce()` 逐帧驱动 UI 与渲染
- 导出了最小 C ABI：
  - `tinaluxAndroidCreateRuntime()`
  - `tinaluxAndroidDestroyRuntime()`
  - `tinaluxAndroidAttachWindow()`
  - `tinaluxAndroidDetachWindow()`
  - `tinaluxAndroidRenderOnce()`
  - `tinaluxAndroidInstallDemoScene()`
- Android 构建下不再生成桌面 `Tinalux` 可执行文件，而是生成共享库 `libtinalux_native.so`
- `GLFW` 只在非 Android 平台接入，避免 Android 目标被桌面窗口库污染
- 新增 JNI 绑定层，默认对接 `com.tinalux.runtime.TinaluxNativeBridge`
- 新增宿主侧 Kotlin 组件：
  - `TinaluxNativeBridge.kt`
  - `TinaluxRendererHost.kt`
  - `TinaluxSurfaceView.kt`
  - `TinaluxInputConnection.kt`
- `AndroidWindow` 现在会在每帧轮询与交换前刷新 `ANativeWindow` 尺寸，便于 surface 改尺寸时触发后续重建
- 新增 Android 触摸到 UI 鼠标事件模型的最小映射：
  - `ACTION_DOWN` -> `MouseButtonPress`
  - `ACTION_MOVE` -> `MouseMove`
  - `ACTION_UP/CANCEL` -> `MouseButtonRelease`
- 新增 Android IME / 键盘桥接能力：
  - 查询 `textInputActive` / `imeCursorRect`
  - `commitText` -> `TextInputEvent`
  - `setComposingText` / `finishComposingText` -> `TextCompositionEvent`
  - 常见 Android key code 映射到现有 `KeyEvent`
- 新增 Android 系统剪贴板桥接：
  - 宿主 `ClipboardManager` 与 native window 内部剪贴板状态双向同步
  - runtime 内部会保留 clipboard 文本，跨 window detach / reattach 不直接丢失
- 宿主 `TinaluxSurfaceView` 新增 `onHostPause()` / `onHostResume()` 包装，
  便于 Activity/Fragment 生命周期先接上帧循环和输入法状态管理
- `pause/resume` 现在不只是宿主层约定，native runtime 也会记录 `suspended` 状态，
  渲染帧循环会在 suspend 期间停止推进
- `Application` 新增渲染级生命周期接口：
  - `suspendRendering()`
  - `resumeRendering(windowConfig)`
  - `renderingReady()`
- Android runtime 在 suspend/reattach 时会保留 UI 会话，释放并重建
  `window/context/surface`，而不是直接把整个 `Application` 关掉
- Android 窗口侧已补上 Vulkan 平台接线：
  - `vkGetInstanceProcAddr`
  - `VK_KHR_surface`
  - `VK_KHR_android_surface`
  - Android Vulkan surface create/destroy
  - 基于临时 surface 的 queue-family present support 查询
- `AndroidRuntimeConfig` 现在默认使用 `Backend::OpenGL`
  作为 Android 宿主默认后端；Vulkan 保持显式启用
- 新增宿主 `TinaluxActivity.kt`，把 `TinaluxSurfaceView` 生命周期接线收成最小基类

## 当前边界

- Android 平台代码只在 `ANDROID` 构建下启用
- 桌面平台行为不变，桌面主入口仍走 `main.cpp`
