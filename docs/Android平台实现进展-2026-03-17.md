## 当前已实现能力

### 窗口与渲染

- `AndroidWindow` 可接收外部注入的 `ANativeWindow`
- 支持 OpenGL ES + EGL 路径
- 支持 Vulkan surface 平台接线
- `Application` 支持 `pumpOnce`
- `Application` 支持 `suspendRendering` / `resumeRendering`

### Runtime 生命周期

- `AndroidRuntime` 可持有 `Application`
- 支持 `attachWindow`
- 支持 `detachWindow`
- 支持 `renderOnce`
- 支持暂停与恢复
- 支持 preferred backend 切换

### 输入与系统桥接

- 触摸到鼠标事件的最小映射
- 文本输入状态查询
- IME 光标矩形查询
- `commitText`
- `setComposingText`
- 常见 Android key code 映射
- 剪贴板桥接

### 宿主层

- `TinaluxSurfaceView`
- `TinaluxRendererHost`
- `TinaluxNativeBridge`
- `TinaluxActivity`
- `TinaluxVulkanValidationActivity`

## 当前验证结论

可以确认的，是“实现链已存在”；不能确认的，是“真机链路已经完整稳定”。

同时，当前脚本与 SDK 构建约束已经补到以下程度：

- `build_android_native.ps1` 支持 `-ValidateOnly`，可先做环境与目录体检
- SDK Gradle 构建会校验目标 ABI，而不是只要任意 ABI 的 so 存在就放行
- Android 验证工程的命令行流程默认依赖本地 Gradle 安装，不再误导为仓库已内置 wrapper

当前更准确的表述：

- Android 已进入实现阶段
- 不是只有架构草案
- 但还需要真机验证和构建收敛

## 当前主要缺口

- 真机编译与运行验证
- OpenGL / Vulkan 两条 Android 后端链路的实际稳定性验证
- 输入法、剪贴板、生命周期在真实设备上的回归验证
- 更完整的移动端交互语义

## 当前建议

- 把本文作为“Android 已实现到哪里”的事实清单
- 不再把 Android 描述为“待实现”
- 但同样不要过度宣称 Android 已经完全可用
