# Tinalux 快速改进指南

> 更新时间：2026-03-18  
> 目标：只保留当前仍然值得立刻执行的事项。

## 1. 先收敛构建和测试目录

- 重新生成构建目录
- 完整构建当前 smoke target
- 用新的 `ctest -N` 核对测试数量

原因：

- 源码里当前定义 `46` 个 smoke target
- 本地 `cmake-build-debug` 只登记到 `44`
- 还缺少不少测试可执行文件

## 2. 优先做 Android 真机验证

建议顺序：

1. 构建 native 库
2. stage 到 `android/tinalux-sdk`
3. 跑 `android/validation-app`
4. 验证 OpenGL 默认后端
5. 验证显式 Vulkan 切换

## 3. 补 Linux/X11 真机闭环

- 先做 `validateTinaGlfwX11Linux.sh`
- 再做实际窗口和 IME 运行验证

当前不要再把 Linux 写成“已完全支持”。
