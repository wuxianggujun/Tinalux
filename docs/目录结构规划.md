## tests 层

源码当前包含五类测试入口：

- `tests/core/`
- `tests/app/`
- `tests/markup/`
- `tests/ui/`
- `tests/scripts/`

源码中的 [`tests/CMakeLists.txt`](../tests/CMakeLists.txt) 当前定义 `76` 个 C++ smoke
target；另外还有 `2` 个 PowerShell 脚本 smoke。
如果本地构建目录里看到的数量不同，应以源码和重新生成后的构建结果为准。

## CMake 现状

### 顶层 `CMakeLists.txt`

当前行为：

- 默认从 `3rdparty/tina_glfw` 引入桌面窗口后端
- 默认从 `3rdparty/spdlog` 引入日志依赖
- 通过 `cmake/SkiaConfig.cmake` 处理 Skia
- 非 Android 构建生成桌面可执行文件 `Tinalux`
- Android 构建生成共享库 `tinalux_native`
- 桌面端可选构建 smoke tests

早期文档里“FetchContent 在线下载 GLFW”这一说法已经过时。

## Android 相关目录关系

```text
android/host            -> 宿主层 Kotlin 组件
android/tinalux-sdk     -> SDK 打包与 Gradle 入口
android/validation-app  -> 对 SDK 和 native 库做最小验证
```

## 当前维护建议

- 结构说明应优先跟 `CMakeLists.txt`、`src/*/CMakeLists.txt` 和 `include/tinalux/*` 保持一致
- 如果再引入新平台目录或工具脚本，应同步更新本文，而不是继续沿用历史规划稿
