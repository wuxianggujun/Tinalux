# Tinalux 项目概述

## 这是什么

Tinalux 是一个从零构建的跨平台 Skia 自绘 UI 框架。每个界面元素——按钮、文本框、面板——都由 Skia 在 GPU 上逐像素绘制，不使用任何操作系统原生控件。

## 为什么做这个

通过亲手实现一个完整的 UI 框架，学习 GUI 底层原理：

- **约束布局算法**：父子组件如何协商尺寸（参考 Flutter BoxConstraints）
- **事件分发模型**：捕获→目标→冒泡三阶段（参考 Chromium/GTK4）
- **Widget 树**：组合模式、递归绘制、脏标记按需重绘
- **文本排版**：字体度量、光标定位、文本选择
- **GPU 渲染管线**：OpenGL 上下文 → Skia GrDirectContext → 帧提交
- **平台抽象**：同一套 Widget 代码跑桌面和移动端

后续目标是集成到自研游戏引擎中作为 UI 模块使用。

## 设计参考

| 参考框架 | 取了什么 |
|---------|---------|
| Flutter / Druid | BoxConstraints 约束布局模型 |
| Chromium / GTK4 | 捕获→目标→冒泡事件分发 |
| LVGL | Widget 直持状态 + 脏标记重绘 |
| Druid | 5 方法 Widget 接口设计 |

## 技术栈

| 组件 | 版本 | 用途 |
|------|------|------|
| C++ | 20 | 语言标准 |
| Skia | Ganesh GL | 2D GPU 渲染引擎 |
| GLFW | 3.4 | 桌面窗口后端（将来 Android NDK） |
| CMake | 3.31+ | 构建系统 |

## 构建

```bash
scripts/syncSkia.sh   # 同步 Skia 源码
cmake -B build -S .    # GLFW 自动下载
cmake --build build
```

## 当前状态（2026-03-14）

**已完成（阶段一~四）**：
- 构建系统（CMake 多模块 + GLFW FetchContent + Skia Debug/Release）
- Platform 模块（Window 接口 + GLFWWindow 完整实现，8 个事件回调，DPI 支持）
- Rendering 模块（GL 上下文创建、窗口 Surface、帧提交，不依赖 Platform）
- UI 模块（Constraints 约束布局、Widget 5 方法基类、Container、VBox/HBox 布局）
- 控件：Label、ParagraphLabel、Panel、Button、Checkbox、Radio、Toggle、Slider、ScrollView、ListView、TextInput
- Theme 系统（dark/light）
- 三阶段事件派发（capture→target→bubble）+ hitTest + 鼠标捕获 + hover 追踪
- 焦点管理（Tab/Shift+Tab 循环 + 点击聚焦）
- 属性动画、剪贴板、多行文本排版（SkParagraph）
- App 模块（Application Pimpl 模式、主循环、脏标记跳帧）
- 8 个 smoke test（version / layout / event / text_input / scroll / controls / animation / paragraph_clipboard）

**下一步（阶段五）**：
- 图片控件
- 脏矩形优化
- 性能缓存（SkSurface / measure / TextMetrics）
