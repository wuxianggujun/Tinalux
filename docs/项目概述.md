# Tinalux 项目概述

## 这是什么

Tinalux 是一个从零构建的跨平台 Skia 自绘 UI 框架。每个界面元素——按钮、文本框、面板——都由 Skia 在 GPU 上逐像素绘制，不使用任何操作系统原生控件。

## 为什么做这个

通过亲手实现一个完整的 UI 框架，学习 GUI 底层原理：

- **约束布局算法**：父子组件如何协商尺寸（参考 Flutter BoxConstraints）
- **事件分发模型**：捕获→目标→冒泡三阶段（参考 Chromium/GTK4）
- **Widget 树**：组合模式、递归绘制、脏标记按需重绘
- **文本排版**：字体度量、光标定位、文本选择
- **GPU 渲染管线**：多后端（OpenGL / Vulkan）→ Skia GrDirectContext → 帧提交 + 自动 fallback
- **平台抽象**：同一套 Widget 代码跑桌面和移动端

后续目标是集成到自研游戏引擎中作为 UI 模块使用。

| 参考框架 | 取了什么 |

## 技术栈

| 组件 | 版本 | 用途 |
|------|------|------|
| C++ | 20 | 语言标准 |
| Skia | Ganesh GL + Ganesh Vulkan | 2D GPU 渲染引擎（多后端） |
| tina_glfw | 基于 GLFW 3.5.0 | 桌面窗口后端 |
| CMake | 3.31+ | 构建系统 |

## 构建

```bash
scripts/syncSkia.sh   # 同步 Skia 源码
cmake -B build -S .    # GLFW 自动下载
cmake --build build
```

## 当前状态（2026-03-16）

**源码核对结论**：
- 事件系统主链路已经完成，核心实现位于 `UIContext`，不是旧文档中的 `Application::dispatchEvent()`
- 公开渲染 API 已收口为 `Canvas / RenderContext / RenderSurface / Image`
- **多后端渲染已实现**：OpenGL + Vulkan 完整可用，含自动 fallback 机制
- Layout、富文本、资源加载、图标系统、离屏渲染、组件级缓存都已具备基础实现

**已完成（阶段一~七的大部分主体工作）**：
- 构建系统（CMake 多模块 + tina_glfw 源码集成 + spdlog FetchContent + Skia Debug/Release）
- Platform 模块（Window 接口 + GLFWWindow 完整实现，含 Vulkan 平台接口、CursorEnter、剪贴板、桌面 IME、DPI 支持）
- Rendering 模块（多后端：OpenGL + Vulkan 完整实现、离屏 Surface、帧提交，不依赖 Platform）
  - `Backend` 枚举（Auto / OpenGL / Vulkan / Metal）+ `createContext()` 统一入口
  - Vulkan：VkInstance 创建、物理设备选择、逻辑设备创建、Swapchain 全生命周期、Semaphore 同步
  - `BackendPlan` 候选排序 + `tryPromoteNextBackend()` 运行时 fallback
- UI 模块（Constraints、Widget 基类、Container、VBox/HBox/Flex/Grid/Responsive 布局、UIContext）
- 控件：Label、ParagraphLabel、RichTextWidget、Panel、Button、Checkbox、Radio、Toggle、Slider、ScrollView、ListView、TextInput、ImageWidget、Dialog、Dropdown、ProgressBar
- Theme 系统（dark/light + ThemeManager + ColorScheme + Typography + Spacing）
- 三阶段事件派发（capture→target→bubble）+ hitTest + 鼠标捕获 + hover 追踪
- 焦点管理（Tab/Shift+Tab 循环 + 点击聚焦）
- 属性动画、剪贴板、多行文本排版（SkParagraph）
- 图片资源抽象（文件加载、内存解码、RGBA 创建、文件缓存、IconRegistry、ResourceManager、ResourceLoader）
- 日志门面（分类、轮转、环境变量配置）+ Debug HUD + 帧统计
- App 模块（Application Pimpl 模式、UIContext 拆分、主循环、脏矩形重绘、Overlay 顶层树）
- Overlay / Dialog 顶层交互能力
- 最小组件级渲染缓存（Panel 可选离屏缓存）
- RuntimeHooks 运行时 hook 机制（窗口/上下文/surface 创建可替换，支持测试注入）
- 当前 `ctest` 已登记 36 个测试条目，全量通过

**当前主线**：

- AndroidWindow 与平台层扩展
- 文本后端进一步抽象与边界收口
