# Tinalux 动画系统使用指南

> 更新时间：2026-03-18  
> 状态：动画调度器已实现，并已接入主循环与部分控件。

## 当前实现结构

动画相关能力分成两层：

- `Animation.h`
  - `AnimationSink`
  - `AnimationHandle`
  - `TweenOptions`
  - `Easing`
- `RuntimeState.h`
  - `AnimationScheduler`

其中 `AnimationScheduler` 是实际调度器，实现了：

- `requestAnimationFrame`
- `animate`
- `cancelAnimation`
- `tick`
- `hasActiveAnimations`
- `nextWakeDelaySeconds`

## 主循环接线

当前不是“动画系统存在但没接入主循环”，而是已经接入：

- `Application` 每帧会调用 `UIContext::tickAnimations(...)`
- `UIContext` 内部驱动 `RuntimeState::animationScheduler`
- 有动画或异步资源回调时会触发重绘需求

## 当前已使用动画的地方

### 控件动画

下列控件已经接入 hover / focus 动画：

- `Button`
- `Checkbox`
- `Radio`
- `Toggle`
- `Slider`
- `TextInput`

其中：

- `Slider` 有 hover + focus 两类动画
- `TextInput` 有 hover + focus 两类动画

### 主题动画

`ThemeManager::setTheme(theme, true)` 会通过 `AnimationSink` 做主题插值动画。

## 基础用法

### 1. 通过 `AnimationSink` 做补间动画

```cpp
using namespace tinalux::ui;

AnimationHandle handle = application.animationSink().animate(
    {
        .from = 0.0f,
        .to = 1.0f,
        .durationSeconds = 0.3,
        .loop = false,
        .alternate = false,
        .easing = Easing::EaseInOut,
    },
    [](float value) {
        (void)value;
        // 根据 value 更新状态
    });
```

### 2. 请求下一帧回调

```cpp
application.animationSink().requestAnimationFrame([](double nowSeconds) {
    (void)nowSeconds;
    // 一次性的下一帧任务
});
```

### 3. 取消动画

```cpp
application.animationSink().cancelAnimation(handle);
```

## 当前适合写进文档的结论

- 动画调度器已经可用
- 动画主循环已经接通
- 主题动画已经可用
- 部分控件动画已经落地

## 当前不应写成“已实现”的内容

- 通用布局动画框架
- 页面导航切换动画框架
- 动画时间轴/编排系统
- 可视化动画调试工具

这些能力目前都还没有形成统一 API。

## 与资源加载的关系

动画和异步资源都由 `UIContext` 驱动：

- `tickAnimations()` 推进动画
- `tickAsyncResources()` 泵送异步图片回调

因此当前重绘触发条件不只来自动画，也可能来自异步资源完成。
