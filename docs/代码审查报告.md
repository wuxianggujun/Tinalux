# Tinalux 代码审查报告

> 更新时间：2026-03-18  
> 说明：早期逐条 bug 清单已经大面积失效。本文只保留当前仍需关注的风险项。

## 当前高优先级风险

### 1. 构建目录与测试清单不一致

当前观察到：

- 源码中定义了 `46` 个 smoke target
- 本地 `cmake-build-debug` 的 `ctest -N` 只列出 `44` 个测试
- 并提示大量测试可执行文件缺失

这不是单纯的文档问题，而是验证基线本身不可靠。

### 2. Android 仍缺真机验证

虽然 Android 的窗口、runtime、JNI、宿主层和验证工程都已经存在，但当前还不能把 Android 状态写成“完全可用”。

主要风险：

- 真机渲染行为
- 输入法行为
- 生命周期恢复
- OpenGL / Vulkan 两条路径的实际稳定性

### 3. Linux/X11 仍缺运行时闭环

Linux/X11 代码链路已经接通，但当前缺的是：

- 真机窗口验证
- IME 真实行为验证
- 不同输入法之间的差异验证

### 4. Metal 后端尚未实现

当前代码里有：

- 枚举
- 候选排序位置
- 相关测试壳

但还没有正式可用的 Metal 渲染后端。

## 中优先级缺口

- 手势 / 触摸语义层
- 无障碍
- Widget Inspector
- 热重载
- 可视化性能分析工具

## 已不再适合作为当前问题单的内容

以下类型的问题，不应继续沿用 2026-03-14 的旧表述：

- 事件系统尚未完成
- Flex / Grid / Responsive 尚未实现
- RichText 尚未实现
- ThemeManager 尚未实现
- ResourceLoader 尚未实现

这些结论现在都和源码不一致。
