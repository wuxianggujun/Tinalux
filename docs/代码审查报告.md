## 一、崩溃级 Bug（P0）

### 1.1 Container::removeChild 解引用已移动的 shared_ptr

**文件**：`src/ui/Container.cpp:20-41`

**问题**：`std::remove_if` 将要保留的元素移动到前面，`[it, end)` 范围内的 `shared_ptr` 已被 move（变为 null）。循环对 null `shared_ptr` 调用 `->setParent()` 会崩溃。

```cpp
// 当前代码 — 有崩溃风险
const auto it = std::remove_if(children_.begin(), children_.end(),
    [child](const std::shared_ptr<Widget>& candidate) {
        return candidate.get() == child;
    });

for (auto current = it; current != children_.end(); ++current) {
    (*current)->setParent(nullptr);  // ⚠️ *current 可能是 null
}
children_.erase(it, children_.end());
```

**复现条件**：删除一个不在末尾的子组件。

**修复方案**：

```cpp
void Container::removeChild(Widget* child)
{
    if (child == nullptr) {
        return;
    }

    auto it = std::find_if(children_.begin(), children_.end(),
        [child](const std::shared_ptr<Widget>& c) { return c.get() == child; });

    if (it != children_.end()) {
        (*it)->setParent(nullptr);
        children_.erase(it);
        markDirty();
    }
}
```

## 二、逻辑 Bug（P1）

### 2.1 Bubble 阶段跳过根节点（off-by-one）

**文件**：`src/app/Application.cpp:97-103`

**问题**：冒泡阶段的循环终止条件 `i-- > 1` 导致 `path[0]`（根节点）不会收到冒泡事件。捕获阶段根节点参与了，但冒泡阶段被跳过，逻辑不对称。

```cpp
// 当前代码
for (std::size_t i = path.size(); i-- > 1;) {
    ui::Widget* current = path[i - 1];
    // ...
}
```

**修复方案**：将 `> 1` 改为 `> 0`。

```cpp
for (std::size_t i = path.size(); i-- > 0;) {
    // path.back() 是 target，从 target 的上一个开始冒泡
    // 但需要跳过 target 自身（已在目标阶段处理）
}
```

注意：需要仔细确认不重复处理 target。当前 bubble 从 `path[size-2]` 开始是为了跳过 target，但停在 `path[1]` 跳过了 root。正确做法是 bubble 从 `path[size-2]` 到 `path[0]`。

### 2.2 键盘事件不走三阶段派发

**文件**：`src/app/Application.cpp:345-347`

**问题**：键盘事件直接发送给 `focusedWidget`，绕过了捕获/冒泡路径。父容器无法在捕获阶段拦截键盘事件。

```cpp
// 当前代码
if (isKeyboardEvent(event.type()) && impl_->focusedWidget != nullptr) {
    event.handled = impl_->focusedWidget->onEvent(event) || event.handled;
}
```

**影响**：未来实现快捷键系统（如根容器拦截 Ctrl+S）时无法工作。

**修复方案**：

```cpp
if (isKeyboardEvent(event.type()) && impl_->focusedWidget != nullptr) {
    dispatchAlongPath(event, impl_->focusedWidget);
}
```

### 2.3 TextInput 失焦后 draggingSelection_ 未重置

**文件**：`src/ui/widgets/TextInput.cpp`

**问题**：拖拽选择文字过程中，窗口失焦（Alt+Tab）。Application 清除了 `focusedWidget` 和 `mouseCaptureWidget`，但 TextInput 的 `draggingSelection_` 仍为 true。窗口重获焦点后，鼠标移动到 TextInput 上会错误地继续拖拽选择。

**修复方案**：覆盖 `setFocused`，在失焦时重置拖拽状态。

```cpp
// TextInput.h — 添加声明
void setFocused(bool focused) override;

// TextInput.cpp
void TextInput::setFocused(bool focused)
{
    if (!focused) {
        draggingSelection_ = false;
    }
    Widget::setFocused(focused);
}
```

注意：需要先将 `Widget::setFocused` 声明为 `virtual`（当前不是 virtual）。

### 2.4 缺少 CursorEnter 回调

**文件**：`src/platform/glfw/GLFWWindow.cpp`

**问题**：未注册 `glfwSetCursorEnterCallback`。鼠标移出窗口时 Application 无法感知，`hoveredWidget` 不会被清理，按钮保持 hover 高亮状态。

**修复方案**：在 GLFWWindow 构造函数中添加 `glfwSetCursorEnterCallback` 注册，在回调中发送 `MouseEnter`/`MouseLeave` 事件。Application 需要在收到窗口级 MouseLeave 时清理 `hoveredWidget`。

## 三、性能问题（P2）

### 3.1 每帧重建 SkSurface

**文件**：`src/app/Application.cpp:408-414`

`renderFrame()` 每次调用都创建新的 `SkSurface`。虽然是轻量包装，但应该缓存并在 resize 时重建。

### 3.2 Layout arrange 阶段重复 measure

**文件**：`src/ui/layouts/VBoxLayout.cpp:66`、`src/ui/layouts/HBoxLayout.cpp:66`

`arrange()` 内部再次对每个子组件调用 `measure()`。measure 阶段的结果未缓存，Widget 树深了之后 measure 调用次数为 O(n²)。

### 3.3 measureTextMetrics 每次创建 SkFont

**文件**：`src/ui/TextPrimitives.h:21`

每次调用 `measureTextMetrics` 都创建新的 `SkFont` 对象。Label/Button/TextInput 在 measure 和 onDraw 中各调一次。

### 3.4 innerExtent 重复定义

**文件**：`src/ui/layouts/VBoxLayout.cpp:13-19` 与 `src/ui/layouts/HBoxLayout.cpp:13-19`

完全相同的函数定义在两个匿名命名空间中。应提取到共享的布局工具函数中。

### 3.5 空闲时 sleep_for(8ms)

**文件**：`src/app/Application.cpp:179`

应使用 `glfwWaitEventsTimeout()` 代替固定 sleep，对笔记本电源效率更友好。

## 四、设计隐患（P3）

### 4.1 EventDispatcher 是死代码

**文件**：`include/tinalux/core/events/EventDispatcher.h`

定义了完整的 `EventDispatcher` 模板类，但项目中没有任何文件使用它。Application 使用 switch 语句直接分发事件。

### 4.2 GLFW 错误回调静默吞掉错误

**文件**：`src/platform/glfw/GLFWWindow.cpp:48`

```cpp
static void onError(int, const char*) {}
```

GLFW 的所有错误被静默丢弃，增加调试难度。建议至少输出到 stderr。

### 4.3 DemoScene lambda 捕获 shared_ptr 生命周期风险

**文件**：`src/ui/DemoScene.cpp:80`

`loginButton->onClick(...)` 的 lambda 捕获了兄弟 Widget 的 `shared_ptr`。如果父容器 `removeChild` 了被捕获的 Widget，该 Widget 的 `parent_` 指针悬空但对象仍存活。当前 Demo 不会触发，但扩展后有风险。

### 4.4 hitTest 坐标约定缺少文档

**文件**：`include/tinalux/ui/Widget.h:25`

`hitTest(float x, float y)` 接受的是本地坐标（相对于 Widget 自身原点），但头文件和实现中均无注释说明此约定。

### 4.5 sendMouseCrossEvent 传递全局坐标

**文件**：`src/app/Application.cpp:108-116`

`MouseEnter`/`MouseLeave` 事件携带的是全局 framebuffer 坐标，而非 Widget 本地坐标。当前不影响功能（Button 只改 `hovered_` 状态，不使用坐标），但语义不正确。
