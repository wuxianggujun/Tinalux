## 一、立即实施（本周内完成）

### 1.1 性能优化 - Surface缓存

**文件**：`src/app/Application.cpp`

**问题**：每帧重建SkSurface，虽然是轻量包装但仍有优化空间

**实施步骤**：

1. 在`Application::Impl`中添加成员变量：
```cpp
rendering::RenderSurface windowSurface_;
int lastFramebufferWidth_ = 0;
int lastFramebufferHeight_ = 0;
```

2. 修改`renderFrame()`方法，添加缓存逻辑：
```cpp
void Application::Impl::renderFrame() {
    int w = window->framebufferWidth();
    int h = window->framebufferHeight();
    
    // 只在尺寸变化时重建surface
    if (!windowSurface_ || w != lastFramebufferWidth_ || h != lastFramebufferHeight_) {
        windowSurface_ = rendering::createWindowSurface(context, w, h);
        lastFramebufferWidth_ = w;
        lastFramebufferHeight_ = h;
    }
    
    // 使用缓存的surface
    auto& canvas = windowSurface_.canvas();
    // ... 后续绘制逻辑
}
```

**预期收益**：减少每帧的surface创建开销

### 1.2 性能优化 - 使用glfwWaitEventsTimeout

**文件**：`src/app/Application.cpp`

**问题**：空闲时使用`sleep_for(8ms)`，对笔记本电源效率不友好

**实施步骤**：

修改主循环：
```cpp
int Application::run() {
    while (!impl_->window->shouldClose()) {
        if (impl_->needsRedraw) {
            impl_->window->pollEvents();
            impl_->renderFrame();
            impl_->needsRedraw = false;
        } else {
            // 使用GLFW的事件等待，最多8ms超时
            glfwWaitEventsTimeout(0.008);
            // 检查是否有事件导致需要重绘
            if (impl_->needsRedraw) {
                impl_->renderFrame();
                impl_->needsRedraw = false;
            }
        }
    }
    return 0;
}
```

**预期收益**：降低CPU占用，提高电源效率

### 2.3 性能优化 - Measure结果缓存

**文件**：`include/tinalux/ui/Widget.h`, `src/ui/Widget.cpp`

**实施步骤**：

1. 在Widget类中添加缓存结构：
```cpp
// Widget.h
protected:
    struct MeasureCache {
        Constraints constraints;
        core::Size result;
        bool valid = false;
        
        bool matches(const Constraints& c) const {
            return valid && 
                   constraints.minWidth() == c.minWidth() &&
                   constraints.maxWidth() == c.maxWidth() &&
                   constraints.minHeight() == c.minHeight() &&
                   constraints.maxHeight() == c.maxHeight();
        }
    };
    
    mutable MeasureCache measureCache_;
```

2. 修改measure方法：
```cpp
// Widget.cpp
core::Size Widget::measure(const Constraints& constraints) {
    if (measureCache_.matches(constraints)) {
        return measureCache_.result;
    }
    
    auto result = onMeasure(constraints);
    measureCache_ = {constraints, result, true};
    return result;
}
```

3. 在markLayoutDirty中失效缓存：
```cpp
void Widget::markLayoutDirty() {
    measureCache_.valid = false;
    // ... 现有逻辑
}
```

**预期收益**：减少重复measure调用，特别是深层Widget树

### 3.5 渲染缓存通用化

**目标**：从Panel扩展到所有Container

**实施步骤**：

1. 在Container基类中添加缓存支持：
```cpp
class Container : public Widget {
public:
    void setCacheEnabled(bool enabled);
    bool cacheEnabled() const;
    
private:
    bool cacheEnabled_ = false;
    rendering::Image cachedImage_;
    bool cacheDirty_ = true;
};
```

2. 修改绘制逻辑：
```cpp
void Container::onDraw(rendering::Canvas& canvas) {
    if (cacheEnabled_ && !cacheDirty_) {
        canvas.drawImage(cachedImage_, 0, 0);
        return;
    }
    
    if (cacheEnabled_) {
        // 绘制到离屏surface并缓存
        auto surface = rendering::createRasterSurface(
            bounds().width(), bounds().height());
        // ... 绘制子组件
        cachedImage_ = surface.snapshotImage();
        cacheDirty_ = false;
    }
    
    // 正常绘制
    drawChildren(canvas);
}
```

3. 添加缓存统计到Debug HUD

## 七、风险和注意事项

### 7.1 兼容性风险

**问题**：新增控件可能依赖尚未实现的Canvas方法

**缓解措施**：
1. 实施前检查Canvas API完整性
2. 必要时先扩展Canvas接口
3. 使用现有API的组合实现复杂图形

### 7.2 性能风险

**问题**：缓存可能导致内存占用增加

**缓解措施**：
1. 设置合理的缓存上限
2. 实现LRU淘汰策略
3. 添加内存监控
4. 提供手动清理接口

### 7.3 API稳定性

**问题**：频繁的API变更影响使用者

**缓解措施**：
1. 遵循语义化版本控制
2. 标记废弃API并保留一段时间
3. 提供迁移指南
4. 在文档中明确标注实验性API

## 八、总结

本实施计划按照优先级将改进工作分为四个阶段：

1. **立即实施**：性能优化和代码质量改进，影响大、成本低
2. **短期实施**：新增核心控件和关键优化，2周内完成
3. **中期实施**：功能完善和架构优化，1个月内完成
4. **长期规划**：平台扩展和架构演进，2-3个月完成

建议严格按照优先级执行，确保每个阶段都有可交付的成果。
